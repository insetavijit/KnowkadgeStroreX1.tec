# Medical Models: AI for Healthcare

Specialized models for medical applications—with caution.

---

## 1. Healthcare AI Applications

| Application | Description |
|-------------|-------------|
| **Diagnosis Support** | Helping clinicians identify conditions |
| **Research** | Analyzing medical literature |
| **Documentation** | Medical transcription and charting |
| **Patient Communication** | Health information in plain language |
| **Drug Discovery** | Analyzing molecular data |

**Important**: Medical AI is assistive, not diagnostic.

## 2. Google Medical Models

**Med-Gemini** (formerly MedPaLM 2):
- Trained on medical textbooks, research, clinical data
- Achieves expert-level on medical exams
- Not publicly available API

**Conceptual usage**:
```python
# When available via API
medical_llm = ChatGoogleGenerativeAI(model="med-gemini")

response = medical_llm.invoke("""
Patient presents with: persistent cough, fatigue, night sweats.
Lab results: [...]

What differential diagnoses should be considered?
""")
```

## 4. Clinical Applications

**Use Cases**:
| Safe | Risky |
|------|-------|
| Literature search | Direct diagnosis |
| Documentation help | Treatment decisions |
| Patient education | Prescribing advice |
| Research analysis | Emergency triage |

**Implementation Pattern**:
```python
class MedicalAssistant:
    """DISCLAIMER: For research/education only."""
    
    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4o")
        
        self.system_prompt = """
You are a medical information assistant.
IMPORTANT:
- You provide general information only
- You are NOT providing medical advice
- Always recommend consulting a healthcare provider
- Do not diagnose or prescribe
"""
    
    def answer(self, question: str) -> str:
        response = self.llm.invoke([
            SystemMessage(content=self.system_prompt),
            HumanMessage(content=question)
        ])
        
        # Add disclaimer
        return f"{response.content}\n\n⚠️ This is general information, not medical advice."
```

## 5. HIPAA Compliance

**Requirements for handling PHI (Protected Health Information)**:

| Requirement | Implementation |
|-------------|----------------|
| **Access Control** | Role-based, audit logs |
| **Encryption** | In transit and at rest |
| **Minimum Necessary** | Only access needed data |
| **Business Associates** | BAA with LLM providers |
| **Audit Trail** | Log all data access |

**Safe Pattern**:
```python
# Don't send PHI to external APIs
# Use de-identified or synthetic data for development

def de_identify(text: str) -> str:
    """Remove PHI before sending to LLM."""
    # Remove names, dates, SSN, MRN, etc.
    return phi_scrubber.scrub(text)

def process_medical_text(text: str):
    clean_text = de_identify(text)
    response = llm.invoke(clean_text)
    return response
```
