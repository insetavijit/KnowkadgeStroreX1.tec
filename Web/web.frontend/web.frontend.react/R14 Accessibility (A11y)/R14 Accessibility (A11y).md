# R14 Accessibility (A11y)

Master accessibility for inclusive React applications. Learn ARIA, keyboard navigation, screen readers, and accessible components. Accessibility knowledge is increasingly required for frontend positions.

|Topic|Focus & Purpose|
|---|---|
|**[[R14.1 A11y Fundamentals]]**|Web accessibility basics; WCAG guidelines; semantic HTML; accessibility tree. A11y concepts.|
|**[[R14.2 Semantic JSX]]**|Semantic elements; heading hierarchy; landmark roles; meaningful structure. Semantic markup.|
|**[[R14.3 ARIA in React]]**|ARIA roles; aria-label; aria-describedby; aria-live; dynamic content. ARIA implementation.|
|**[[R14.4 Keyboard Navigation]]**|Focus management; tab order; keyboard shortcuts; skip links; focus trapping. Keyboard users.|
|**[[R14.5 Screen Reader Testing]]**|Testing with VoiceOver/NVDA; announcing changes; accessible forms; common issues. Screen readers.|
|**[[R14.6 Accessible Components]]**|Accessible modals; accessible dropdowns; accessible tabs; ARIA patterns. Component a11y.|
|**[[R14.7 A11y Testing]]**|axe-core; eslint-plugin-jsx-a11y; automated testing; manual testing; audit tools. A11y testing.|
|**[[R14.8 Project: Accessible App]]**|Building WCAG compliant app; keyboard testing; screen reader testing; audit. A11y project.|
