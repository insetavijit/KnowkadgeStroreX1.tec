# 07 Routing with React Router

Master client-side routing for single-page applications. Learn React Router v6, nested routes, protected routes, and navigation patterns. Routing is essential for all multi-page React applications.

|Topic|Focus & Purpose|
|---|---|
|**[[7.1 Router Basics]]**|React Router setup; BrowserRouter; Routes; Route; path matching; index routes. Routing foundation.|
|**[[7.2 Navigation]]**|Link component; NavLink; useNavigate; programmatic navigation; navigation state. Page navigation.|
|**[[7.3 Route Parameters]]**|Dynamic segments; useParams; URL parameters; optional parameters; parameter validation. Dynamic routes.|
|**[[7.4 Nested Routes]]**|Nested route structure; Outlet component; layout routes; relative paths; shared layouts. Route nesting.|
|**[[7.5 Protected Routes]]**|Auth guards; route protection; redirect logic; role-based access; loading states. Secure routing.|
|**[[7.6 Search & State]]**|Query parameters; useSearchParams; location state; URL state management. URL-based state.|
|**[[7.7 Data Loading]]**|Loaders; data fetching per route; error handling; suspense integration; loading UI. Route data.|
|**[[7.8 Project: Multi-Page App]]**|Building SPA; nested routing; protected routes; data loading; navigation. Routing project.|
