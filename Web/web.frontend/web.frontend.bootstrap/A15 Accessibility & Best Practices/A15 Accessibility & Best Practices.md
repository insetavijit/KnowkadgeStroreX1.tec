# A15 Accessibility & Best Practices

Master Bootstrap accessibility for inclusive web development. Learn ARIA, keyboard navigation, screen readers, and semantic HTML. Build applications usable by everyone. Accessibility knowledge is increasingly required for frontend positions.

|Topic|Focus & Purpose|
|---|---|
|**[[A15.1 Semantic HTML]]**|Semantic elements in Bootstrap; landmark roles; heading hierarchy; meaningful structure; component semantics. Structural accessibility.|
|**[[A15.2 ARIA Attributes]]**|ARIA roles; aria-label; aria-describedby; aria-hidden; aria-expanded; live regions. Screen reader support.|
|**[[A15.3 Keyboard Navigation]]**|Focus management; tab order; keyboard shortcuts; skip links; focus indicators; focus trapping. Keyboard users.|
|**[[A15.4 Color Contrast]]**|WCAG contrast requirements; checking contrast; accessible color choices; text on images; testing tools. Visual accessibility.|
|**[[A15.5 Form Accessibility]]**|Label association; required fields; error messages; input instructions; form validation accessibility. Accessible forms.|
|**[[A15.6 Screen Reader Testing]]**|Testing with NVDA/VoiceOver; common issues; accessible components; testing workflow; bug fixes. Testing accessibility.|
|**[[A15.7 Performance Best Practices]]**|CSS optimization; unused CSS removal; PurgeCSS; lazy loading; critical CSS; performance testing. Speed optimization.|
|**[[A15.8 Project - Accessible Website]]**|Building A11y-compliant site; WCAG 2.1 AA; keyboard testing; screen reader testing; audit. Accessibility showcase.|
